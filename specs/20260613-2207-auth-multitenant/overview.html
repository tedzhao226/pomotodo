  • history: get_all_tasks_with_stats, get_completed_blocks_page

EDUCATION Why JWKS, not a shared secret

Source: EDUCATION.md. The token's signature is what turns an opaque string into a trusted user_id. Who can verify decides who can forge.

Per-user? The token is per-user (each login mints a fresh JWT whose sub = that user's id); the signing key is one per project, shared across all users. Signature proves authenticity (shared key); sub provides identity (per user) and becomes user_id. A sub can't be changed without breaking the signature — so the shared key does not weaken isolation.

Glossary

subSubject — the Supabase user id; this app's user_id.
aud / issAudience / issuer — checked to reject a token from another project (e.g. dev replayed on prod).
kidKey id in the token header; selects which JWKS public key verifies it.
JWKSJSON Web Key Set — Supabase endpoint publishing the current public keys.
